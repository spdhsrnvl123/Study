# -p 옵션
# 경로에 있는 폴더 전부 생성 + 이미 있으면 에러 안냄
mkdir -p /var/www/chatbot

# 경로에 따른 Linux 관례
# 웹사이트 파일(Nginx, Apache)
/var/www
# 사용자 개인 파일
/home/사용자
# 추가 설치 프로그램
/opt
# 설정파일
/etc

# Nginx 설정
```
server {
    listen 80;
    server_name chatbot.con.or.kr;  # 너 도메인으로 변경

    root /var/www/chatbot;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```
# Nginx 재시작
systemctl reload nginx

# Nginx 설정 파일 구조
/etc/nginx/
├── nginx.conf              ← 메인 설정 파일
├── conf.d/                 ← 추가 설정 폴더
│   └── chatbot.conf        ← 챗봇 전용 설정
└── ssl/                    ← SSL 인증서


DNS란?
Domain Name System = 도메인 이름을 IP주소로 변환해주는 시스템
사용자가 chatbot.con.or.kr 입력
         ↓
DNS 서버에 물어봄: "이 도메인 IP가 뭐야?"
         ↓
DNS 응답: "211.115.xxx.x이야"
         ↓
해당 IP 서버로 접속
         ↓
Nginx가 server_name 보고 처리
